~ (Promise<Instance>) findById(id)

Locate a single instance by id.

Parameters:

  • id ( String | Number ) — The id of the Instance as understood by the backing store.

Returns:

  • ( Promise<Instance> ) — A Promise of the Instance with the given id, if found. If not found, the Promise will resolve with the value undefined. The Promise is only rejected in the event of a database error.

~ (Promise< Array<Instance> >) findById(ids)

Locate multiple instances given a list of ids.

Parameters:

  • id ( Array<String | Number> ) — The ids of the Instances as understood by the backing store.

Returns:

  • ( Promise< Array<Instance> > ) — A Promise of an array whose entries correspond to the entries of the ids array. In each position, the array will contain the Instance with the given id, if found. If not found, the entry will be undefined. The Promise is only rejected in the event of a database error.

# (Promise<Instance>) find(options)

Retrieve a single instance from the backing store using query options.

Parameters:

  • options ( Object ) — Query options. NB Not all backends need support all options.

Returns:

  • ( Promise<Instance> ) — A Promise of the Instance matching the query, if found. If not found, the Promise will resolve with the value undefined. The Promise is only rejected in the event of a database error.

# (Promise<Results>) findAll(options)

Retrieve many instances from the backing store using query options.

Parameters:

  • options ( Object ) — Query options. NB Not all backends need support all options.

Options Hash: (options):

  • offset ( Number ) — Offset for pagination.
  • limit ( Number ) — Limit (number of entries per page) for pagination.
  • cursor ( Cursor ) — A Cursor object returned by a previous findAll operation. If passed, this operation will retrieve the next page. NB Passing a pagination object may override other query options in an attempt to match your query against the one that generated the pagination.

Returns:

  • Promise<Results>
