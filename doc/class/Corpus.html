Class: Corpus

Defined in: src/Corpus.coffee

Overview

A collection of models connected to a collection of backends.

Instance Method Summary

  • # (void) createModel(spec) Create a model within this Corpus with the given spec.
  • # (Model) getModel(name) Get the Model with the given name.
  • # (Backend) getBackend(name) Get the Backend with the given name.
  • # (void) getDefaultBackendForModel(model) Get the default backend for the given model.
  • # (void) bindAllModels() Attach all models to default bindings.

Constructor Details

# (void) constructor(config = {})

Create a new Corpus

Parameters:

  • config ( Object ) — Configuration data.

Options Hash: (config):

  • promise ( Object ) — Promise implementation to use internally. By default, uses whatever Promise implementation is installed in the global Promise variable. The object is of the form js { reject, resolve, all }
  • backends ( Object<String, Backend> ) — A hash of named backends available to this Corpus.
  • log ( Object ) — A bunyan-style logger object of the form js {trace, debug, info, warn, error, fatal}. All ormojo logging will be directed through this object. NB Trace-level logging is extremely verbose!
  • defaultBackend ( String ) — The name of the backend to use by default when an operation would require a named backend but none is specified.
